## Design Context

### Users
The primary user is the repository owner using the site as a personal culinary lab: browsing canonical Markdown recipes, cooking from phase-based procedures, and iterating on technical details over time. The interface should support quick scanning in a kitchen or desk context without turning the notes into a separate CMS.

### Brand Personality
Clinical, precise, uncompromising. The site should feel like a Modernist Cuisine-inspired lab instrument: authoritative, structured, and optimized for exact recipe data rather than casual storytelling.

### Aesthetic Direction
Primary theme is a light laboratory card system based on the Stitch Modernist recipe design system. Use stark white and tinted neutral surfaces, crisp black text, thin rules, square corners, dense tables, and rare Modernist red accents. Avoid warm editorial styling, rounded cards, decorative shadows, and magazine-like food-blog composition.

### Typography System
Use Space Grotesk for display, headline, and technical labels. Display type should feel architectural and oversized, with `display-lg` at 3.5rem used for section headers and recipe/name entries where the layout can support it. Headlines and navigation are all-caps Space Grotesk. Labels use `label-sm` at 0.6875rem for metadata such as dates, tags, phase labels, and annotations, including rotated metadata where it reinforces the grid. Body copy uses Work Sans `body-lg` at 1rem to keep recipe prose readable and human against the colder display system.

### Design Principles
- Markdown remains canonical; the website is only a precise presentation layer over the repository.
- Recipe pages should prioritize phase order, ingredient quantities, scaling, structural notes, and failure modes.
- Use flat hierarchy: 1px rules, tonal surfaces, restrained red emphasis, and no ornamental depth.
- Favor dense, scan-friendly layouts over explanatory marketing sections.
- Keep interactions quiet, rectangular, and predictable so the content feels like technical documentation.
